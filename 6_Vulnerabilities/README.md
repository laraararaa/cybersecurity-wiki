# 🐛 Vulnerabilities & Exploits

A **vulnerability** is a weakness in a system that can be exploited. An **exploit** is code or technique that takes advantage of a vulnerability.

## Vulnerability Lifecycle

```
Vulnerability Discovered
    ↓
Vendor Notified (Responsible Disclosure)
    ↓
Vendor Develops Patch
    ↓
Patch Released
    ↓
Companies Apply Patch (or don't!)
    ↓
Vulnerability Either Fixed or Exploited
```

## Common Vulnerability Sources

- **Design flaws**: Architecture is inherently insecure
- **Coding errors**: Mistakes in code
- **Misconfiguration**: System setup is insecure
- **Outdated software**: Old versions have known vulnerabilities
- **Zero-days**: Unknown vulnerabilities no one has patched yet

## Vulnerability Severity

- **Critical**: Immediate, remote exploitation, high impact
- **High**: Serious impact, difficult exploitation
- **Medium**: Moderate impact or difficult exploitation
- **Low**: Minor impact, very difficult exploitation

## Why Understand Vulnerabilities?

- **For attackers**: Find and exploit weaknesses
- **For defenders**: Identify and patch weaknesses
- **For everyone**: Understand risks
